# Pythia

A RAG (retrieval-augmented generation) service for WG21 C++ proposals. Search
across a corpus of ISO C++ papers by semantic meaning, filter by section type
or paper status, and explore the citation graph between proposals.

## Stack

| Layer | Technology |
|---|---|
| Embeddings | `sentence-transformers` (`all-MiniLM-L6-v2`) |
| Vector store | Qdrant (Docker) |
| Reranking | Cohere (`rerank-english-v3.0`) |
| Citation graph | NetworkX |
| API | FastAPI |
| UI | Streamlit |

## Corpus

20 papers in `data/papers/`:

| Paper | Topic |

## Setup

**Requirements:** Python 3.14+, Docker

```bash
uv sync
docker compose up -d
```

Set your Cohere API key to enable reranking (optional — falls back to vector
score order without it):

```bash
export COHERE_API_KEY=<your-key>
```

## Ingest

Parse, chunk, embed, and index all papers (~5–10 minutes):

```bash
python ingest.py
```

This populates the `wg21_papers` Qdrant collection and writes `graph.json`.
Re-running is safe — the collection is wiped and rebuilt from scratch each time.

## Run

Start the API:

```bash
uvicorn main:app --reload
```

In a separate terminal, start the UI:

```bash
streamlit run app.py
```

API: `http://localhost:8000` — Swagger UI at `http://localhost:8000/docs`
UI: `http://localhost:8501`

Swagger UI is available at `http://localhost:8000/docs`.

## API

### `POST /search`
Semantic search with optional filters and Cohere reranking.

```json
{
  "query": "sender/receiver async model",
  "filters": {
    "section_type": "motivation",
    "status": "accepted"
  },
  "top_n": 5
}
```

Supported filter keys: `status`, `section_type`, `author`, `date_from`, `date_to`.

### `GET /papers/{paper_id}`
Metadata and citation neighbours for a single paper (e.g. `/papers/P2300R7`).

### `GET /graph/{paper_id}`
Citation graph for a paper — cites / cited-by, with boundary nodes flagged.
Boundary nodes are papers referenced in the corpus but not indexed.

### `GET /corpus`
All indexed papers with metadata.

## UI

Three tabs:

- **Search** — run a query, apply sidebar filters, browse expandable result cards
- **Paper** — look up a paper by ID; view metadata and citation table with
  boundary nodes highlighted in red
- **Corpus** — table of everything in the index
